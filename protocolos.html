Tema 2 - HTTPS / DNS

Una Introducción a HTTPS y DNS

HTTPS (Protocolo de Transferencia de Hipertexto Seguro) es la versión segura de HTTP, utilizada para transferir datos entre un navegador y un sitio web de manera encriptada, lo que aumenta la seguridad en la comunicación de información.

El uso de HTTPS es especialmente importante en sitios que requieren credenciales de inicio de sesión. En navegadores modernos, como Chrome, se destaca visualmente cuando un sitio no utiliza HTTPS, advirtiendo al usuario sobre la falta de seguridad en la conexión.

Solicitudes GET y POST

  • GET
    • Se utiliza para solicitar datos de un servidor sin modificarlo. Es un método idempotente, lo que significa que realizar la misma solicitud varias veces no debe causar efectos secundarios en el servidor.
  • POST
    • Se utiliza para enviar datos al servidor para que este los procese, como en el caso de enviar información a una base de datos o realizar una acción que cambie el estado del servidor.

Diferencias entre HTTP y HTTPs

  • HTTP
    • Protocolo básico de comunicación para la transmisión de datos entre un navegador y un servidor. Se usa principalmente para la carga de páginas web y la transferencia de datos.
  • HTTPs
    • Es una versión segura de HTTP, que utiliza encriptación para proteger los datos transmitidos entre el cliente y el servidor. Necesita obtener un certificado SSL/TLS de una autoridad de certificación (CA). El servidor luego utiliza este certificado para establecer conexiones seguras con los navegadores de los usuarios.

Códigos de estado

  • 1XX
    • Mensaje informativo.
  • 2XX
    • Éxito.
      • 200: OK
      • 201: Creado
      • 202: Aceptado
      • 204: Sin contenido
  • 3XX
    • Redirección.
      • 300: Múltiples opciones
      • 301: Movido permanentemente
      • 302: Encontrado
      • 304: No modificado
  • 4XX
    • Error del cliente.
      • 400: Solicitud incorrecta
      • 401: No autorizado
      • 403: Prohibido
      • 404: No encontrado
  • 5XX
    • Error del servidor.
      • 500: Error interno del servidor
      • 501: No implementado
      • 502: Puerta de enlace incorrecta
      • 503: Servicio no disponible

DNS

Su función es resolver nombres (dominios) legibles para las personas en direcciones IP legibles para las máquinas. Utiliza una base de datos distribuida y jerárquica que almacena información asociada a nombres de dominio

  • Funcionamiento
    • Los servidores DNS que reciben la petición, buscan si disponen de la respuesta en la memoria caché.
    • Si lo encuentran, sirven la respuesta.
    • En caso contrario, inician la búsqueda en otros DNS.
    • Encontrada la respuesta, el servidor guarda el resultado en su memoria para futuros usos y devuelve el resultado.
  • Reenviadores
    • Consultas a nombres DNS externos a servidores DNS normalmente fuera de la red.

Resoluciones directa e inversa

Su función es resolver nombres (dominios) legibles para las personas en direcciones IP legibles para las máquinas. Utiliza una base de datos distribuida y jerárquica que almacena información asociada a nombres de dominio

  • Resolución directa
    • Permite traducir un nombre a una dirección IP.
  • Resolución inversa
    • Mediante la definición de registros PTR permite obtener el nombre que corresponde con una dirección IP.
    • Se basa en el dominio in-addr.arpa.
    • Se van añadiendo los diferentes números en la IP en sentido inverso.
